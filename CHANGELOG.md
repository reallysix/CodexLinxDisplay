# 更新日志

## 0.3.2

- 将 Codex 顶部安全区与自定义图片顶部留白拆分为独立设置，自定义图片默认全屏显示。
- 多图切换间隔、Codex 顶部安全区和图片顶部留白支持直接输入数值，同时保留步进箭头。
- 手动输入的数值会自动限制在有效范围内。

## 0.3.1

- 将动态宽度的 SwiftUI 菜单栏场景改为固定方形的 AppKit 状态图标，避免更新重启后图标因菜单栏空间不足而消失。
- 新版本首次启动时自动显示设置窗口，明确提示应用已成功重启。
- 再次打开已在后台运行的应用时主动显示设置窗口，避免用户没有可见入口。
- 增加菜单栏状态项与版本升级可见性的回归测试。

## 0.3.0

- 自定义图片模式支持一次选择多张图片并定时轮播。
- 支持顺序与随机切换，随机模式不会连续重复当前图片。
- 增加“填充裁剪”和“完整显示”两种图片适配方式。

## 0.2.0

- 增加 Codex 用量与自定义图片两种显示模式。
- 图片模式会暂停 Codex 用量读取，切回 Codex 时立即刷新并推送。
- 自定义图片自动裁切为 142×428，并保留顶部状态栏安全区。

## 0.1.1

- 修复无 Developer ID 签名的预览安装包启动后立即退出的问题。
- 增加发布构建签名检查，防止 adhoc 预览包错误启用 Hardened Runtime。

## 0.1.0

- 读取本机 Codex 周用量、可用重置次数与自动重置时间。
- 生成适配 Linx68 142×428 屏幕的中文 JPEG，并为顶部状态栏保留安全区。
- 支持自定义图片 API、定时后台推送、登录时启动与立即测试。
- 支持应用内检查更新与安装新版本。
